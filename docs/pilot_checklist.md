# Pilot Checklists

Use the following checklists to coordinate preparations and retrospectives for ChessGuard pilot events. Duplicate this file per event or track status in your project management tool while referencing these canonical items.

## Pre-Event Readiness Review

### Governance & Planning
- [ ] Pilot charter approved with success metrics, scope, and exit criteria.
- [ ] Federation agreement and legal approvals signed, including data processing addenda.
- [ ] Roles assigned (Pilot Lead, Analysts, Technical Support, Communications, Legal).
- [ ] Runbook, SOPs, and escalation paths reviewed by all stakeholders.

### Technical Preparation
- [ ] Ingestion pipelines validated with sample telemetry.
- [ ] Detection engine thresholds configured for event format and player pool.
- [ ] Analyst console deployed to staging and smoke-tested.
- [ ] Alert routing, notifications, and logging verified end to end.
- [ ] Backup and recovery procedures tested (database snapshots, queue replay).

### Security & Privacy
- [ ] Access control list reviewed; least privilege enforced for all accounts.
- [ ] MFA verified for analysts and administrators.
- [ ] Consent language distributed to registration/communications teams.
- [ ] Data retention schedule confirmed and documented.

### Operations & Logistics
- [ ] Training sessions delivered to analysts and arbiters.
- [ ] Communications templates customized with event-specific details.
- [ ] Daily briefing cadence scheduled with required participants.
- [ ] Incident response channel tested (chat, bridge, paging).
- [ ] Player device checks and hardware requirements communicated.

### Go/No-Go Confirmation
- [ ] Dry-run exercise completed with simulated alerts.
- [ ] Outstanding risks documented with mitigation owners.
- [ ] Pilot Lead issues formal go/no-go decision.

## Post-Event Retrospective

### Immediate Wrap-Up (T+0 to T+1)
- [ ] All alerts closed or handed off with clear owners.
- [ ] Final determinations communicated to federation leadership.
- [ ] Sensitive data exports secured or deleted per retention policy.
- [ ] Incident log and escalation summaries completed.

### Stakeholder Feedback
- [ ] Collect analyst survey responses and debrief notes.
- [ ] Gather player/arbitration feedback (surveys, interviews, support tickets).
- [ ] Meet with federation counterparts to review satisfaction and concerns.
- [ ] Compile technical support issues and resolutions.

### Metrics & Reporting
- [ ] Produce pilot summary report (alert volumes, outcomes, time-to-disposition).
- [ ] Analyze model performance (precision, recall, false positive rate) and propose tuning actions.
- [ ] Update risk register with new findings and control effectiveness.

### Continuous Improvement
- [ ] Prioritize improvement backlog items and assign owners with due dates.
- [ ] Update runbook, SOPs, and knowledge base articles with lessons learned.
- [ ] Review security/privacy posture and document remediation plans.
- [ ] Confirm next steps for broader rollout or subsequent pilots.

### Closure
- [ ] Pilot Lead signs off on completion and archives documentation.
- [ ] Legal/compliance verifies retention and deletion tasks complete.
- [ ] Communications lead sends thank-you or closing message to stakeholders.
